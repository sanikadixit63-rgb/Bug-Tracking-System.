# Users and Permissions

This document defines the types of users in the Bug Tracking System and their permissions.

## Admin
Admin is responsible for managing bugs and overseeing the system.

Admin can:
- Create new bugs
- View all bugs in the system
- Assign bugs to developers
- Update the status of any bug

Admin cannot:
- Work on bugs as a developer

## Developer
Developer is responsible for fixing assigned bugs.

Developer can:
- View bugs assigned to them
- Update the status of assigned bugs

Developer cannot:
- Create new bugs
- Assign bugs to other users
- View bugs assigned to other developers
