# Bug Data and Lifecycle

This document explains what information a bug contains and how it progresses in the system.

## Bug Data
Each bug in the system contains the following information:
- Title: Short summary of the issue
- Description: Detailed explanation of the problem
- Status: Current state of the bug
- Assigned Developer: Developer responsible for fixing the bug
- Created Date: When the bug was reported

## Bug Status Values
A bug can have one of the following statuses:
- Open: Bug has been created but work has not started
- In Progress: Bug is actively being worked on
- Closed: Bug has been fixed and completed

## Bug Lifecycle
1. Admin creates a bug with status set to Open.
2. Admin assigns the bug to a Developer.
3. Developer starts working on the bug and updates status to In Progress.
4. Developer completes the fix and updates status to Closed.
5. Closed bugs remain visible for tracking and reference.
